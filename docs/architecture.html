terraform-gcp-artifact-registry Architecture Document

Artifact Registry Module

Executive Summary

A production-ready Terraform module for provisioning and managing Google Artifact Registry repositories. Supports nine package formats (Docker, Maven, NPM, Python, APT, YUM, Go, Kubeflow, Generic) with three repository modes (Standard, Virtual, Remote). Includes automated cleanup policies, granular IAM bindings, CMEK encryption via Cloud KMS, and VPC Service Controls integration for enterprise-grade artifact management.

Component Breakdown

google_artifact_registry_repository.this

Primary repository resource supporting all nine package formats. Dynamically configures Docker immutable tags, Maven version policies, Virtual upstream routing, and Remote proxy/cache to public registries (Docker Hub, Maven Central, NPMJS, PyPI) or custom URIs.

google_artifact_registry_repository_iam_member.bindings

Flattened IAM bindings allowing role-based access grants (reader, writer, admin) at the repository level using for_each iteration over role-member pairs.

Cleanup Policies

Dynamic cleanup policy blocks supporting DELETE and KEEP actions with conditions based on tag state, prefixes, age thresholds, and most-recent-version retention counts. Dry-run mode available for safe validation.

Data Flow

  • Push: CI/CD pipelines authenticate via IAM, push artifacts (images, packages) to the Standard repository endpoint.
  • Pull (Standard): Consumers pull artifacts directly using the repository URL ({location}-docker.pkg.dev/{project}/{repo}).
  • Pull (Virtual): Requests route through upstream policies, aggregating artifacts from multiple backing repositories by priority.
  • Pull (Remote): Acts as a caching proxy, fetching from upstream registries (Docker Hub, Maven Central, NPMJS, PyPI, or custom URIs) and caching locally.
  • Cleanup: Automated policies delete untagged or aged artifacts and retain the most recent N versions per package.

Industry Adaptation

  • Financial Services: CMEK encryption + VPC SC for regulatory compliance, immutable Docker tags for audit trails.
  • Healthcare: Private repository with strict IAM and cleanup policies for PHI-adjacent container images.
  • E-Commerce: Virtual repositories aggregating internal and third-party packages; Remote mode caching public registries to reduce build latency.
  • Enterprise DevOps: Multi-format repositories (Docker + Maven + NPM) with automated lifecycle management.

Production Readiness Checklist

  • Enable CMEK encryption with a dedicated Cloud KMS key
  • Configure VPC Service Controls perimeter
  • Set cleanup policies (delete untagged after 30d, keep latest 5 versions)
  • Enable Docker immutable tags for container repositories
  • Assign least-privilege IAM bindings (avoid allUsers / allAuthenticatedUsers)
  • Test cleanup policies with cleanup_policy_dry_run = true before enforcement
  • Apply consistent labels for cost tracking and resource organization
  • Document repository naming conventions and ownership

Deployment

module "artifact_registry" {
  source = "github.com/kogunlowo123/terraform-gcp-artifact-registry"

  project_id    = "my-project"
  location      = "us-central1"
  repository_id = "my-docker-repo"
  format        = "DOCKER"
  kms_key_name  = "projects/my-project/locations/us-central1/keyRings/my-ring/cryptoKeys/my-key"

  docker_config = { immutable_tags = true }

  cleanup_policies = [
    {
      id     = "delete-untagged"
      action = "DELETE"
      condition = { tag_state = "UNTAGGED", older_than = "2592000s" }
    },
    {
      id     = "keep-latest"
      action = "KEEP"
      most_recent_versions = { keep_count = 5 }
    }
  ]

  iam_bindings = {
    "roles/artifactregistry.reader" = ["serviceAccount:ci@my-project.iam.gserviceaccount.com"]
    "roles/artifactregistry.writer" = ["serviceAccount:cd@my-project.iam.gserviceaccount.com"]
  }

  labels = { team = "platform", env = "production" }
}
